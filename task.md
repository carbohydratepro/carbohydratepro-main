1. [ ] 家計簿CSV一括インポート機能（クレカ・PayPay・銀行明細対応）
    ## 背景
    クレジットカード・PayPay・銀行のAPI連携は電子決済等代行業（金融庁登録）が必要なため個人開発では現実的でない。
    各社が提供するCSV明細ダウンロードを活用した「CSVアップロード→一括登録」が最も現実的・合法な手段。
    ## 対応する連携サービス
    - **楽天カード**: 楽天e-NAVIからCSVダウンロード可能（確定分のみ）
    - **三井住友カード**: VpassからCSVダウンロード可能（PC版のみ）
    - **JCB**: MyJCBからCSVダウンロード可能
    - **American Express**: ログイン後の明細ダウンロード対応
    - **PayPay**: PayPayアプリ→履歴→CSVエクスポートから取得可能
    - **各銀行（みずほ・UFJ・ゆうちょ等）**: 銀行によりCSV明細ダウンロード対応
    ## 実装内容
    1. **CSVアップロードUI**
       - 家計簿画面に「CSV一括インポート」ボタンを追加
       - ファイル選択後、文字コード（UTF-8/Shift-JIS）を `chardet` で自動判定して読み込む
    2. **列マッピング設定画面**
       - CSVの列（日付・金額・摘要等）をどのフィールドに対応させるか設定する画面
       - 設定を「プリセット」として保存（例：「楽天カード」「PayPay」等のプリセット名で登録）
       - 一度登録したプリセットは次回から自動適用
    3. **プレビュー・確認画面**
       - インポート前にパース結果を一覧表示し、誤認識があれば修正できる
       - カテゴリ・支払い方法の自動割当（摘要の文字列から推定）
       - 重複チェック（同日・同金額・同摘要のトランザクションが既に存在する場合に警告）
    4. **一括登録処理**
       - 確認後に一括でTransactionモデルへ保存
       - 失敗した行はスキップしてエラー一覧を表示
    ## 技術スタック
    - バックエンド: `csv`（Python標準）, `chardet`（文字コード自動判定）, `python-dateutil`（日付形式自動解析）
    - フロントエンド: TypeScript（`src/ts/csv_import.ts` を新規追加）
    - 新規テンプレート: `app/templates/app/expenses/csv_import.html`
    - 新規URL: `/carbohydratepro/expenses/import/`
    ## 各社CSVフォーマット例（プリセット）
    | サービス | 日付列 | 金額列 | 摘要列 | 備考 |
    |---|---|---|---|---|
    | 楽天カード | 利用日 | 利用金額(税込) | 利用店名・商品名 | Shift-JIS |
    | PayPay | 取引日時 | 金額 | 相手の名前 | UTF-8 |
    | 三井住友カード | ご利用日 | ご利用金額 | ご利用店名 | Shift-JIS |
    | みずほ銀行 | 取引日 | お引出し/お預入れ | 摘要 | Shift-JIS |

1. [ ] Suica利用履歴インポート（PaSoRiカードリーダー対応）
    ## 背景・制約
    - JR東日本はSuicaの公式APIを一般開発者に公開していない
    - スマートフォンのブラウザ（Web NFC API）はFeliCa規格（交通系ICカード）に非対応のため直接読み取り不可
    - **PaSoRi（ソニー製FeliCaリーダー、3,000〜5,000円）＋WebUSB API** を使ってPC ChromeブラウザからSuicaの利用履歴を直接読み取ることは技術的に可能
    - Suicaの利用履歴データは暗号化なしで読み取り可能（最大20件）
    ## 実装内容
    1. **PaSoRi接続UI**
       - 家計簿画面またはインポート画面に「SuicaをPaSoRiで読み取る」ボタンを追加（PC Chrome専用）
       - WebUSB API非対応ブラウザでは非表示またはグレーアウト＋「Chrome（PC版）が必要です」メッセージ
       - ボタンクリック→ブラウザのUSBデバイス選択ダイアログ→PaSoRiを選択→Suicaカードをかざすよう案内
    2. **FeliCa読み取り処理**（`src/ts/suica_reader.ts` を新規追加）
       - Sony公式の「Web Client SDK for NFC/FeliCa」または `muojp/webpasori`（MIT License）を使用
       - サービスコード `0x090F`（利用履歴）で最大20件のデータを読み取り
       - 各レコード（16バイト）を以下にデコード：
         - 利用年月日（バイト0-1）
         - 利用種別・入出場区分（バイト2-3）
         - 路線コード・駅コード（バイト4-7）→駅名マスターで変換
         - 残額（バイト10-11）
         - 取引通し番号（バイト13-14）
    3. **プレビュー・登録画面**
       - 読み取ったデータを一覧表示（日付・摘要・金額）
       - 金額は「前回残高 - 今回残高」で計算（差分）
       - カテゴリを「交通費」に自動設定、支払い方法を「Suica」に自動設定
       - ユーザーが修正して一括登録
    4. **重複防止**
       - 取引通し番号をDBに保存し、同じ取引の重複登録を防ぐ
    ## 技術スタック
    - フロントエンド: `src/ts/suica_reader.ts`（WebUSB API + Sony SDK or webpasori）
    - バックエンド: 読み取りはフロントで完結。データ送信→Transactionモデルへ保存
    - 駅名マスター: 国土交通省の鉄道駅データ or `ekidata.jp` のCSVを参照
    ## 制約・注意事項
    - **対応環境**: PC版 Chrome/Edge のみ（WebUSBはFirefox/Safari未対応、モバイルChromeも不可）
    - **必要機器**: PaSoRi RC-S380/S390（ユーザーが別途購入）
    - **履歴件数**: カード上に保存されているのは最新20件のみ（FeliCaの仕様）
    - **対応カード**: Suica・PASMO・ICOCA等の交通系ICカード全般に対応見込み
    - UI上で「PaSoRiをお持ちの方向けの機能です」と明示する

1. [ ] Zaim API連携（オプション：Zaimユーザー向け取引インポート）
    ## 背景
    Zaimは個人開発者向けの公開OAuth APIを提供している日本では数少ない家計簿サービス。
    Zaimにすでに取引を登録しているユーザーが本アプリにデータをインポート・同期できるようにする。
    ## 実装内容
    1. **OAuth認証フロー**
       - マイページに「Zaimと連携する」ボタンを追加
       - Zaim開発者ポータル（https://dev.zaim.net）でアプリ登録→Consumer Key/Secret取得
       - OAuth 1.0aフロー: `https://api.zaim.net/v2/auth/request`→ユーザー認証→コールバックでアクセストークン取得
       - トークンを暗号化してDB保存（`ZaimToken` モデルを新規追加）
    2. **取引データ取得**
       - `GET https://api.zaim.net/v2/home/money` で取引データ取得（最大100件/回、ページネーション対応）
       - 支出（payment）・収入（income）・振替（transfer）をTransactionモデルに変換してインポート
       - カテゴリ名のマッピング（Zaim側のカテゴリ名 → 本アプリのCategoryモデルへ変換）
    3. **重複防止**
       - Zaim側の `id` フィールドを `Transaction.external_id` として保存し重複インポートを防ぐ
       - 差分インポート（前回インポート日時以降のデータのみ取得）
    4. **手動同期ボタン**
       - マイページの「Zaim連携」セクションに「今すぐ同期」ボタンを配置
       - 同期ログ（最終同期日時・取得件数）を表示
    ## 技術スタック
    - バックエンド: `requests-oauthlib`（OAuth1.0a対応）, 新規 `ZaimToken` モデル
    - 新規URL: `/carbohydratepro/integrations/zaim/`
    - 注意: Zaim APIはOAuth **1.0a**（2.0ではない）のため `requests-oauthlib` の `OAuth1Session` を使用
