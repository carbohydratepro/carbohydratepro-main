# リリース前 E2E 自動化仕様

`test.md` の確認項目のうち、まず Playwright に落とすスモークテストを定義します。未自動化の項目は `test.md` のチェックリストとして残し、必要に応じてこのファイルへケース化して追加します。

<a id="e2e-auth-001"></a>
## E2E-AUTH-001 ログインできる

- 対応元: `test.md` 1. 認証・アカウント
- 前提データ: `E2E_USER_EMAIL` と `E2E_USER_PASSWORD` で指定した有効ユーザーが存在する。
- 手順: `/login/` を開き、メールアドレスとパスワードを入力し、ログインボタンを押す。
- 期待値: `/carbohydratepro/expenses/` へ遷移し、家計簿画面とログイン済みナビが表示される。
- 自動化: `e2e/tests/auth.spec.ts`
- 手動確認: メール認証状態、アカウント停止状態、複数ロールの差分。

<a id="e2e-auth-002"></a>
## E2E-AUTH-002 ログイン失敗時に分かりやすいエラーが表示される

- 対応元: `test.md` 1. 認証・アカウント
- 前提データ: なし。
- 手順: `/login/` を開き、存在しないメールアドレスと任意の誤ったパスワードでログインする。
- 期待値: `メールアドレスとパスワードが一致しません。` が表示され、ログイン画面に留まる。
- 自動化: `e2e/tests/auth.spec.ts`
- 手動確認: 未入力時のブラウザ/サーバー側必須エラー。

<a id="e2e-auth-004"></a>
## E2E-AUTH-004 サインアップ時にパスワード不一致エラーが表示される

- 対応元: `test.md` 1. 認証・アカウント
- 前提データ: なし。テストごとに未使用のユーザー名とメールアドレスを生成する。
- 手順: `/signup/` を開き、ユーザー名、メールアドレス、異なる2つのパスワードを入力して送信する。
- 期待値: `/signup/` に留まり、確認用パスワード欄がエラー表示になり、パスワード不一致の文言が表示される。
- 自動化: `e2e/tests/auth.spec.ts`
- 手動確認: 登録成功時の確認メール、重複メール、弱いパスワード、メール認証完了フロー。

<a id="e2e-acc-001"></a>
## E2E-ACC-001 アカウントを追加して相互に切り替え、解除できる

- 対応元: `test.md` 1. 認証・アカウント（複数アカウント切替）
- 前提データ: `E2E_USER_EMAIL`/`E2E_USER_PASSWORD`（メイン）と `E2E_SECONDARY_USER_EMAIL`/`E2E_SECONDARY_USER_PASSWORD`（サブ）の2ユーザーが存在し、互いに未連携である。
- 手順: メインでログインし、ヘッダーのアカウントメニューから「アカウント追加」を開き、サブの認証情報で追加する。アカウント編集ページで現在アカウントを確認し、メインへパスワードなしで切り替える。最後にサブの連携を解除する。
- 期待値: 追加後はサブへ切り替わり「切り替えました」が表示される。編集ページに両アカウントが表示され、切替後はメインが「現在のアカウント」になる。解除後はサブが一覧から消える。
- 自動化: `e2e/tests/account-switch.spec.ts`
- 手動確認: アイコン画像アップロード、孫アカウント（サブの他の連携先）が一覧に露出しないことのUI確認。

<a id="e2e-acc-002"></a>
## E2E-ACC-002 現在のアカウントのみログアウトすると連携アカウントへ自動切替する

- 対応元: `test.md` 1. 認証・アカウント（複数アカウント切替）
- 前提データ: E2E-ACC-001 と同じ2ユーザー。
- 手順: メインでログインしてサブを追加（現在=サブ）。アカウントメニューの「ログアウト」を押す。編集ページで状態を確認後、サブを解除して完全にログアウトする。
- 期待値: サブのみログアウトされメインへ自動切替し、「現在のアカウントからログアウトし、〜に切り替えました」が表示される。編集ページでサブに「ログアウト済み」バッジが付く。最後のログアウトでログイン画面へ遷移する。
- 自動化: `e2e/tests/account-switch.spec.ts`
- 手動確認: ログアウト済みアカウントへの「ログインして切り替え」でメール入力済みログイン画面になること。

<a id="crud-and-api"></a>
## 主要 CRUD・API

- 前提データ: `seed_e2e_user` で作成したテストユーザー、支払方法、カテゴリ、メモ種別が存在する。
- 共通期待値: 作成した `e2e-*` データが一覧に表示され、テスト内で削除される。テスト用ユーザー以外の既存データには依存しない。

| ID | 手順 | 期待値 |
| --- | --- | --- |
| `E2E-EXPENSES-CRUD-001` | 家計簿一覧で収支登録モーダルを開き、日付、金額、用途、取引タイプ、費用タイプ、カテゴリ、支払方法を入力して登録し、検索して削除する。 | 登録後に検索結果へ用途が表示され、削除後は一覧から消える。 |
| `E2E-RECURRING-CRUD-001` | 定期支払い新規登録画面で用途、金額、分類、支払方法、頻度を入力して登録し、有効状態を切り替えて削除する。 | 登録後に定期支払い一覧へ用途が表示され、切替POSTが成功し、削除後は一覧から消える。 |
| `E2E-MEMO-CRUD-001` | メモ作成モーダルでタイトル、種別、本文、お気に入りを入力して登録し、検索、お気に入り切替、削除を行う。 | 検索結果へタイトルが表示され、切替POSTが成功し、削除後は一覧から消える。 |
| `E2E-SHOPPING-CRUD-001` | 買い物追加モーダルで一時買い物を登録し、検索、購入済み切替、削除を行う。 | 検索結果へタイトルが表示され、購入済み切替POSTが成功し、削除後は一覧から消える。 |
| `E2E-TASK-CRUD-001` | タスク登録モーダルで終日タスクを登録し、日表示へ移動して削除する。 | 対象日のガント表示にタイトルが表示され、削除POSTが成功する。 |
| `E2E-HABIT-CRUD-001` | 習慣一覧で習慣追加モーダルを開き、毎日の良い習慣を登録し、ダッシュボードで達成切替して削除する。 | 習慣一覧にタイトルが表示され、達成切替POSTが成功し、削除後は一覧から消える。 |
| `E2E-BOARD-CRUD-001` | 一時タスクボードAPIでセット作成、カード作成、カード更新、カード削除、セット削除を行う。 | 各APIが成功ステータスを返し、作成したセットとカードを削除できる。 |

- 自動化: `e2e/tests/crud.spec.ts`
- 手動確認: ダブルクリック/ダブルタップ編集、長押し削除、ドラッグ移動、細かいモーダル操作。

## 自動化しない/別系統で確認する項目

- 端末依存: iOS Safari、Android Chrome 実機、タッチ長押し/スワイプの体感。
- 目視品質: 色、余白、固定フッター被り、細かい崩れ、グラフの見やすさ。
- 運用環境: 本番 `DEBUG=False` の 500 ページ、サーバーログ、メール配送、cron 実行。
- 高負荷: 大量データ性能、キャッシュ、ヒートマップ大量 DOM。
