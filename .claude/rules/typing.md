## 型定義のルール

### Python

- `Any` 型の使用は極力避けること
- `Unknown` 型が推論される場合は明示的に型を指定すること
- 関数の引数と戻り値には必ず型アノテーションを付けること
- `# type: ignore` コメントは最終手段としてのみ使用すること
- Django のモデルフィールドには適切な型ヒントを使用すること（django-stubs を参照）

### JavaScript

- TypeScript を使用する場合は `any` 型の使用を避けること
- `unknown` 型を使用し、適切な型ガードで絞り込むこと
- JSDoc コメントで型情報を提供することを推奨
