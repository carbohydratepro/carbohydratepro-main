## UI/UX 系

- [ ] **ダークモード対応** `難易度: 低` `優先度: A`
  - **市場動向**: 2024年時点で主要生産性アプリのほぼ100%がダークモードに対応。ユーザーの約45%がダークモードをデフォルト設定で利用。システム設定に追従する自動切替も標準的に期待される。
  - **実装方法**:
    1. CSS変数で全カラーを定義し `@media (prefers-color-scheme: dark)` でオーバーライド
    2. `[data-theme="dark"]` 属性方式でJS手動切替と組み合わせ
    3. 2024年全主要ブラウザ対応済みの `light-dark()` CSS関数も活用可能
    4. `localStorage` に設定を保存し、ページ読み込み時に即座に適用（フラッシュ防止）
  - **注意点**: 既存の `styles.css` にカラー変数が定義されていれば改修コストが下がる。画像・アイコンのカラー反転にも注意。



- [ ] **ボトムナビゲーション（モバイル）** `難易度: 低` `優先度: A`
  - **市場動向**: 2024-2025年のモバイルUXトレンドは「ボトムナビゲーション優勢」。Redbootがボトムナビ導入でDAU +65%・セッション時間 +70%の改善報告あり。ハンバーガーメニューより最大20-30%操作が高速。推奨パターンは主要5機能をボトムタブ、設定・補助機能はサイドドロワー。
  - **実装方法**:
    1. モバイル（`max-width: 768px`）時のみ表示するCSS `position: fixed; bottom: 0`
    2. 5機能（家計簿・タスク・メモ・買い物・習慣）＋マイページアイコン
    3. 現在の画面をアクティブ状態で強調表示
    4. Safe Area（iPhone Xのホームインジケーター）対応に `env(safe-area-inset-bottom)` を使用
    5. PC時は既存の上部ナビゲーションを継続利用

- [ ] **モーダル・画面遷移アニメーション** `難易度: 低` `優先度: B`
  - **市場動向**: 2025年時点でChrome・Edge・Safari 18+・Firefox 144+が View Transitions API をサポート（Baseline Newly Available）。追加ライブラリなしでネイティブなページ遷移アニメーションが実現できる時代に。
  - **実装方法**:
    1. モーダルの開閉: CSS `transition` + `transform: scale(0.95)→scale(1)` と `opacity` を組み合わせる
    2. 画面遷移: `document.startViewTransition(() => ...)` でSPA風のスライドイン/アウト
    3. フォールバック: API未対応ブラウザでは通常の更新にフォールバック
  - **実装量**: `modal.css` と `app.ts` に各20〜30行追加程度

- [ ] **長押し＋スワイプアクション（モバイル）** `難易度: 中` `優先度: A`
  - **市場動向**: iOS標準のスワイプ削除（メールアプリ等）・長押しコンテキストメニューはユーザーの強い期待値になっている。task.mdにも「ダブルタップ編集・長押し削除」の要件あり。
  - **実装方法**:
    1. **長押し**: `touchstart` で500ms後に `contextmenu` イベント発火（編集・削除ボタンをポップアップ表示）
    2. **スワイプ**: `touchstart/touchmove/touchend` でdeltaXを計算し、閾値（要素幅の20%）超えで操作確定
    3. 左スワイプ→削除確認（赤背景）、右スワイプ→編集
    4. `pointer-events: none` でスクロールと競合しないよう制御
  - **適用画面**: memo/list, shopping/list, task/list
  - **ライブラリ**: Vanilla JS（`src/ts/app.ts` に共通ユーティリティとして実装）

- [ ] **キーボードショートカット対応（PC）** `難易度: 低` `優先度: B`
  - **市場動向**: Linear・Notion・Todoist など生産性アプリのヘビーユーザーはキーボード操作を強く好む。LinearはKeyboard Shortcutsを主要差別化要素として明記。
  - **実装方法**:
    1. `document.addEventListener('keydown', handler)` で共通リスナー
    2. `n`: 各画面の新規作成モーダルを開く
    3. `Escape`: モーダルを閉じる（既存実装があれば確認・統一）
    4. `?`: ショートカット一覧モーダルを表示
    5. フォーム入力中は無効化（`e.target.tagName === 'INPUT'` でスキップ）

- [ ] **Empty State（空状態）デザイン改善** `難易度: 低` `優先度: A`
  - **市場動向**: NN/g（Nielsen Norman Group）がEmpty Stateをオンボーディングの最重要接点と定義。初回ユーザーへの誘導・コンバージョン改善に直結する。アイコン＋タイトル＋説明文＋CTAボタンの4点セットが標準構成。
  - **実装方法**:
    1. 全一覧画面（expenses, tasks, memos, shopping, habits）にEmpty Stateを実装
    2. SVGアイコン or 絵文字＋メッセージ＋「最初の○○を追加する」ボタン
    3. ユーザー名を含めたパーソナライズメッセージで親しみやすさを演出
    4. 「初回」「完了後（全タスク終了等）」「エラー」「検索結果なし」の4パターンで分ける
  - **適用**: 全 `list.html` テンプレートの `{% if not xxx %}` ブロックを刷新

- [ ] **ドラッグ&ドロップ並び替え（共通化）** `難易度: 中` `優先度: A`
  - **市場動向**: カンバンボードは既にDnD実装済み。買い物リスト・メモ・習慣一覧でも手動並び替えの需要が高い。共通ライブラリに抽出して再利用することで開発コストを削減できる。
  - **実装方法**:
    1. HTML5 Drag and Drop API（`draggable="true"`, `dragover`, `drop`イベント）がシンプルで外部依存なし
    2. タッチ対応が必要な場合は `interact.js` を検討
    3. ドロップ後に `order` フィールドをDB更新するAPIエンドポイントを各モデルに追加
    4. `src/ts/app.ts` に共通の `initSortable(container, onSort)` 関数を実装
  - **DBへの影響**: ShoppingItem・Memo・Habitモデルに `order: IntegerField` を追加

- [ ] **アクセシビリティ対応** `難易度: 中` `優先度: B`
  - **市場動向**: WCAG 2.1 AA準拠はグローバルスタンダード。SEOにも間接的に影響。スクリーンリーダー対応・キーボード操作完全対応により障害を持つユーザーにも提供可能に。
  - **実装方法**:
    1. 全インタラクティブ要素に `aria-label`・`aria-describedby` を追加
    2. モーダルに `role="dialog"` と `aria-modal="true"`、フォーカストラップを実装
    3. 色だけでなくアイコン・テキストでも状態を伝える（色覚障害対応）
    4. フォームバリデーションエラーを `aria-invalid` + `aria-errormessage` で連携

## 家計簿系

- [ ] **予算管理機能（カテゴリ別・袋分け）** `難易度: 中` `優先度: S`
  - **市場動向**: マネーフォワードME・Zaim・YNAB・Monarch Moneyいずれも予算管理は中核機能。日本では「袋分け家計簿」として文化的に浸透しており、カテゴリ別予算の月次残額表示が最も直感的なUI。YNABは「すべてのドルに仕事を与える」ゼロベース予算で世界的支持を得ている。
  - **実装方法**:
    1. `Budget` モデルを新規追加: `{user, period(YYYY-MM), category, allocated_amount, rollover_amount}`
    2. 支出一覧画面の上部に「今月の予算 vs 実績」プログレスバーを各カテゴリに表示
    3. 使用率80%超で黄色、100%超で赤色警告（予算超過アラートとも連携）
    4. 設定画面からカテゴリ別に月予算を設定できるUI
    5. 翌月への繰り越し or リセットを選択可能にする
  - **DB変更**: `Budget` モデルと migration を新規追加

- [ ] **CSVインポート/エクスポート** `難易度: 中` `優先度: S`
  - **市場動向**: 日本の銀行・クレカ明細のCSVダウンロードは標準的に提供されており、ユーザーから最も要望の多い機能の1つ。API連携（電子決済等代行業登録必須）の代替として高い実用性がある。各機関でフォーマットが異なるため「列マッピング方式」が最も現実的。
  - **実装方法**:
    1. **エクスポート**: 期間・フィルタ条件を指定してTransaction一覧をCSVダウンロード。`csv.DictWriter` で日付/金額/カテゴリ/支払い方法/摘要列を出力
    2. **インポート**: Shift-JIS・UTF-8を自動判定（`chardet` ライブラリ）。日付フォーマット（YYYY/MM/DD等）も自動解析
    3. **列マッピング画面**: CSVアップロード後、各列をどのフィールドに対応させるか設定する画面。一度設定したプリセットを保存できる（銀行ごとにプリセット登録）
    4. インポート確認画面でプレビュー表示してから確定
  - **ライブラリ**: Python標準 `csv`, `chardet`（文字コード判定）, `python-dateutil`（日付解析）

- [ ] **レシートOCR自動入力** `難易度: 中` `優先度: A`
  - **市場動向**: 2025年のトレンドは「LLM Vision API直接解析」。Google Cloud Vision API + ルールベース処理（精度80-90%）より、Gemini 2.0 Flash（6,000枚/1ドル）やGPT-4o Visionの方がレイアウト多様性・低品質画像への対応で優位。Dr.Wallet（日本）は人間オペレーター併用で精度99.98%を標榜。
  - **実装方法**:
    1. スマホカメラから画像アップロード（`<input type="file" accept="image/*" capture="environment">`）
    2. バックエンドでGemini 2.0 Flash Vision APIに送信: プロンプトで `{date, store, items:[{name, price}], total}` のJSONを指定出力
    3. 抽出結果を「支出登録フォームに自動入力」→ユーザーが確認・修正して確定
    4. 合計金額の整合性チェック（合計 ≒ 各行の合計）でバリデーション
  - **ライブラリ**: `google-generativeai` Python SDK
  - **注意点**: 長いレシートは複数枚撮影→結合処理も検討

- [ ] **家計簿グラフ・可視化強化** `難易度: 低` `優先度: A`
  - **市場動向**: 支出の「見える化」は家計簿アプリの根幹。Copilot・マネーフォワードいずれも多角的なグラフ表示を重要機能として位置づけ。以下の3機能を1タスクとしてまとめて実装効率を上げることを推奨。
  - **統合すべき機能**:
    1. **カテゴリ別支出の円グラフ（Donutチャート）**: 月次表示画面にカテゴリ割合を可視化。Chart.js の `doughnut` タイプで実装
    2. **前月比較表示**: 各カテゴリの今月 vs 先月の増減を矢印アイコン＋差額で表示。`selectors.py` に前月同カテゴリのクエリを追加
    3. **支払い方法別集計**: 現金・クレジット・電子マネー別の合計を棒グラフで表示

- [ ] **定期支払い画面の強化** `難易度: 低` `優先度: A`
  - **市場動向**: サブスクリプションの管理は2024-2025年の家計簿アプリで最も注目される機能の1つ。「気づかず払い続けているサブスク」の可視化ニーズが高い。
  - **統合すべき機能**:
    1. **次回実行日の表示**: `RecurringPayment.should_execute_on()` ロジックを逆算し、一覧に「次回: ○月○日」列を追加
    2. **年間コスト表示**: 各定期支払いの年間総額を計算して一覧に表示。全件合計の「年間固定費合計」も表示
    3. **サブスクリプション検出**: Transactionの摘要フィールドをパターン分析し、定期的な同額支払いを自動検出して「定期支払いに登録しますか？」と提案

- [ ] **収支サマリーウィジェット（ダッシュボード向け）** `難易度: 低` `優先度: A`
  - **市場動向**: マネーフォワード・Mint後継各アプリ全てがトップ画面に今月の収支サマリーを配置。ダッシュボードを実装する際の基礎コンポーネント。
  - **実装方法**: `selectors.py` の `get_summary()` を活用してウィジェット用JSON APIを追加。今月収入・支出・残高・前月比を返す。

- [ ] **取引の一括操作（複数選択・一括削除・一括編集）** `難易度: 低` `優先度: B`
  - **市場動向**: CSVインポート後の整理など、大量データ操作の需要がある。
  - **実装方法**:
    1. 一覧の各行にチェックボックスを追加、全選択トグルも配置
    2. 選択数が1以上の場合、下部に「選択中の○件を削除」「一括カテゴリ変更」バーを表示
    3. JS側でDOMのチェック状態を管理、APIエンドポイントにIDリストをPOST

- [ ] **取引の分割登録（按分機能）** `難易度: 中` `優先度: B`
  - **市場動向**: 飲食費の食費/交際費按分、割り勘精算など。家族・カップル利用で特に需要。
  - **実装方法**:
    1. Transaction登録フォームに「この金額を分割する」ボタンを追加
    2. N件の按分トランザクションを別々のカテゴリ・金額で登録（合計が元金額と一致するようバリデーション）

## タスク管理系

- [ ] **サブタスク・チェックリスト** `難易度: 低` `優先度: A`
  - **市場動向**: Todoist・Notion・TickTickがいずれもサブタスクを標準装備。2024-2025年のTodoist AIアシスタントはタスクをサブタスクに自動分解する機能を追加した。完了率のプログレスバー表示はユーザーモチベーションを向上させる。
  - **実装方法**:
    1. `SubTask` モデルを新規追加: `{task(FK), title, is_completed, order}`
    2. タスク詳細モーダルにチェックリスト形式のサブタスク入力欄を追加（JS でDOMに動的追加）
    3. サブタスク完了率（完了数/総数）をプログレスバーでタスクカードに表示
    4. サブタスク全完了→親タスクを自動完了に変更するオプション

- [ ] **タスク × カレンダー連携（段階実装）** `難易度: 中→高` `優先度: A`
  - **市場動向**: 「タスクとカレンダーの統合」は2025年の最重要ニーズ。Googleが2025年11-12月にGoogle CalendarとGoogle Tasksの相互連動を正式リリース。TickTick・Akiflowはアプリ内統合で差別化。段階的実装が現実的。
  - **段階1（低難易度・先行実装）: iCal形式エクスポート**
    1. `icalendar` ライブラリで `.ics` ファイルを生成するエンドポイントを追加（`pip install icalendar django-ical`）
    2. ユーザーはこのURLをGoogleカレンダー/iCalendar/Outlookに購読登録できる（ReadOnly）
    3. 実装コスト: 0.5〜1日
  - **段階2（中難易度）: Google Calendar API 双方向同期**
    1. OAuth2フロー（`google-auth-oauthlib`）でユーザーの認証情報を取得
    2. タスク作成/更新/削除時にGoogleカレンダーへイベントをCRUD
    3. リフレッシュトークンを暗号化してDB保存
    4. 実装コスト: 3〜5日

- [ ] **ポモドーロ/Flowtimeタイマー** `難易度: 低` `優先度: A`
  - **市場動向**: TickTickが内蔵ポモドーロタイマーを主要差別化機能として訴求。2025年トレンドは固定25分から**Flowtime法**（集中が切れたタイミングで休憩を取る方式）へのシフト。ユーザーに選択肢を提供するのが最良。
  - **実装方法**:
    1. タスクカード/モーダルに「タイマー開始」ボタンを追加
    2. `Vanilla JS` のタイマー実装（`setInterval`）。ポモドーロモード（25/5分固定）とFlowtimeモード（任意開始・任意停止）を切り替え可能に
    3. 作業セッションの時間を `WorkSession` モデルに記録（任意）
    4. ブラウザタブが非アクティブでもタイマー継続（`Web Workers`）
  - **ライブラリ**: 外部依存なし

- [ ] **タスクアーカイブ機能** `難易度: 低` `優先度: A`
  - **市場動向**: 完了タスクが一覧に溜まると視認性が下がる。Things3は「Logbook」としてアーカイブを実装し好評。
  - **実装方法**:
    1. `Task` モデルに `is_archived: BooleanField` を追加（migration）
    2. アーカイブしたタスクは通常一覧から除外、アーカイブ画面で確認
    3. 「完了から30日後に自動アーカイブ」オプションも追加
    4. アーカイブ画面から「復元」「完全削除」が可能

- [ ] **カンバンボード強化（ラベル・期限・メモ）** `難易度: 低` `優先度: B`
  - **市場動向**: Trelloは2024年にAIアシスタントとカードカバー画像・カラーラベルを強化。シンプルなカンバンでも視覚情報の充実が差別化につながる。
  - **統合すべき機能**:
    1. **ラベル対応**: `TempTaskItem` に `label: CharField` と `label_color: CharField` を追加。カードにカラーバッジを表示
    2. **期限設定**: `due_date: DateField` を追加。期限切れカードを赤枠表示
    3. **メモ**: `description: TextField` を追加。カードをクリックで詳細モーダル展開

- [ ] **繰り返しタスクのスキップ機能** `難易度: 低` `優先度: B`
  - **市場動向**: 旅行・体調不良等による一時的なスキップニーズは高い。習慣トラッカーのストリーク凍結と同じ概念。
  - **実装方法**:
    1. タスクの繰り返しオカレンスを特定日だけスキップするためのスキップ日付リストを `skipped_dates: ArrayField` で持つ
    2. スキップした日には「スキップ済み」バッジを表示

- [ ] **タスクのスター・ピン機能** `難易度: 低` `優先度: B`
  - **実装方法**: `is_pinned: BooleanField` を追加。ピン済みタスクを一覧上部に固定表示。ヘッダー/フッターに「ピン留め」アイコンボタンを追加。

- [ ] **タスクの見積もり時間と統計** `難易度: 低` `優先度: B`
  - **市場動向**: Reclaim・Motionが「タスクの工数見積もりから自動スケジューリング」を実装して急成長。見積もり時間の入力だけでも、週単位の作業量把握に役立つ。
  - **実装方法**:
    1. `estimated_minutes: IntegerField` を `Task` モデルに追加
    2. タスク一覧の統計エリアに「今週の予定作業時間: ○時間」を表示
    3. 「今週完了タスク数」「今月完了タスク数」も合わせて表示

- [ ] **タスクのコメント機能** `難易度: 中` `優先度: B`
  - **市場動向**: Jira・Linear・Notionのタスクコメントは開発者に普及。個人用途でも作業ログ・メモとして利用価値が高い。
  - **実装方法**:
    1. `TaskComment` モデルを追加: `{task(FK), content, created_at}`
    2. タスク詳細モーダルにコメント入力欄と履歴を追加
    3. マークダウン対応（`markdown-lite.ts` を再利用）
