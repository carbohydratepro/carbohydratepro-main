1. [ ] **スケルトンスクリーン** `難易度: 低` `優先度: A`
  - **市場動向**: Notion・Linearなど主要アプリが全画面でスケルトンUIを採用。スピナーよりも体感速度が早く、CLSを防止する効果もある。
  - **実装方法**:
    1. CSS `shimmer` アニメーション（`background: linear-gradient(90deg, ...)`）で実装
    2. CSS `:empty` 擬似クラスを活用するとJSゼロで自動切替できる
    3. 各一覧画面（expenses/list, task/list, memo/list等）のロード中に表示
    4. 実際のコンテンツレイアウトを模したスケルトン形状が効果的
  - **ライブラリ**: Pure CSS（外部依存なし）

1. [ ] **トースト通知の統一** `難易度: 低` `優先度: A`
  - **市場動向**: バラバラなフラッシュメッセージはUXの一貫性を損ない、ユーザーの信頼感が下がる。Django `messages` framework の出力を統一的なトーストコンポーネントに変換するだけで大きく改善。
  - **実装方法**:
    1. `base.html` の messages ブロックを統一デザインのトーストに変更
    2. 成功（緑）/エラー（赤）/警告（黄）/情報（青）の4種アイコン付き
    3. 3〜5秒で自動消去・手動クローズ対応
    4. 画面右下 or 上中央に固定表示
  - **ライブラリ**: Vanilla JS実装（外部ライブラリ不要）


## 習慣トラッカー系

1. [ ] **ストリーク表示と最長記録** `難易度: 低` `優先度: S`
  - **市場動向**: Duolingo・Streaks・Habitifyいずれもストリークを中核機能として採用。ストリーク7日以上で日次利用頻度が2.3倍になるとのデータあり（損失回避心理の活用）。ストリーク単体よりマイルストーンと組み合わせると DAU 40〜60%向上。
  - **実装方法**:
    1. `Habit` モデルに `current_streak: IntegerField` と `longest_streak: IntegerField` を追加
    2. `HabitRecord` 保存時に連続日数を計算・更新するシグナルまたはメソッドを実装
    3. 習慣カードに炎アイコン🔥と連続日数をバッジ表示
    4. 最長記録を更新したときに「新記録達成！」のトースト通知を表示
